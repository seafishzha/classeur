<h1 id="writing">Writing</h1>
<hr>
<h2 id="ten-tips-for-writing-cs-papers"><a href="http://www.nowozin.net/sebastian/blog/ten-tips-for-writing-cs-papers-part-1.html">Ten Tips for Writing CS Papers</a></h2>
<ol>
<li>simple language:
<ul>
<li>short sentences</li>
<li>clear logical</li>
<li>minimal jargon.</li>
</ul>
</li>
<li>state your contribution:
<ul>
<li>insight:</li>
<li>performance:</li>
<li>capability:</li>
</ul>
</li>
<li>See everything as a facet on the contribution
<ul>
<li>introduction: needness for the contribution</li>
<li>related work: differentiation with previous works</li>
<li>method: description with formulation</li>
<li>experiments: verification with progaraming</li>
</ul>
</li>
<li>Consider Using a Page-1 Figure
<ul>
<li>simplicity: understand less than 20 seconds</li>
<li>self-contained: all information</li>
</ul>
</li>
<li>Avoid the Passive Voice
<ul>
<li>easier to read</li>
<li>more engaging</li>
</ul>
</li>
<li>Ideal Structure of a Paragraph
<ul>
<li>one paragraph one idea</li>
<li>begin and end glue the context</li>
</ul>
</li>
<li>Avoid Ambiguous Relative Pronouns (This, These, That, Which)
<ul>
<li>add the noun</li>
</ul>
</li>
<li>Provide Continuation Markers
<ul>
<li>navigation</li>
</ul>
</li>
<li>Multiple Authors</li>
<li>Authorship and Author Ordering</li>
</ol>
